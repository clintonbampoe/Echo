# Administrator

This role has the highest level of permissions in a church instance. The administrator is responsible for governance, global configuration and final approvals.

---

- **Limit**
  - **Maximum accounts**: 1 account per instance -- single owner for clear accontabilty

- **Purpose and Accountability**
  - Owns system governance and global configurations for the instance. Accountable to the governance board (this is to state that the admin of the system, accounts to the leaders/governance board in the physical church)

- **Write permissions(Modules)**
  - Membership Management
  - Core Accounting
  - Contributions
  - Projects
  - Assets
  - Attendance
  - Reporting

- **Responsibitlities**
  - Create, disable and transfer user accounts and role assignments. Document transfers and reasons
  - Configure global system settings
  - Maintain governance controls
  - Approve high-risk governance changes and oversee emergency delegations (only in specific situations)
  - Review and approve actions and operations made by roles with less permissions

- **Acceptance Criteria**  
  - Admin can perfrom and persist all global configuration changes via UI
  - UI shows clear warnings for Admin-only changes and requires explicit confirmation for high-risk changes.
  - Admin actions are logged for auditing by church governance.
